Dependencies

If you plan to install RoboViz from source (recommended), the following components should be installed:

  • Java Development Kit (JDK)
  • Git (optional)
  • Apache Ant
Git is only necessary if you want to clone the repository; you can download the source as a .zip file instead. Ant is a build tool that replaces shell scripts and batch files from the previous version. I suggest you use the Oracle JDK if possible (or Apple's JDK on OS X), as I have not tested the pure open-source alternatives.

Install from Source

First, clone the git repository containing the latest source code by entering the following into a shell:

$ git clone --recursive git://github.com/jstoecker/roboviz.git
Next, move into the root roboviz directory and compile the project:
$ cd roboviz
$ ant
If successful, you should see a bin/ directory that contains roboviz.jar and a lib/ subdirectory with other libraries. You can move the contents of the bin/ directory anywhere you want; it does not reference any files in the root roboviz directory.

Starting RoboViz

You can start RoboViz by executing roboviz.jar. From a terminal, you can do the following:

$ java -jar roboviz.jar
Some environments may allow you to double-click on roboviz.jar to launch it.
